# 图

## 定义
图由顶点集合和边集合组成，可以表示多对多关系。
图可以分为有向图、无向图、带权图等。

图就像是一张社交网络关系图：
- **节点（顶点）**：社交网络中的每个人
- **边**：人与人之间的关系（好友、关注等）
- **权重**：关系的强度（亲密度、互动频率等）

现实生活案例：
- 地图：城市是节点，道路是边，距离是权重
- 电路：元件是节点，导线是边，电阻是权重
- 互联网：网站是节点，链接是边，流量是权重
- 交通网络：车站是节点，线路是边，票价是权重

## 原理

### 基本术语
- **顶点（Vertex）**：图中的基本单元，社交网络中的人
- **边（Edge）**：连接两个顶点的线，两人之间的关系
- **度（Degree）**：顶点连接的边数，一个人的好友数量
- **路径（Path）**：顶点的序列，从 A 到 B 的路线
- **环（Cycle）**：起点和终点相同的路径
- **连通图**：任意两点间都有路径
- **权重（Weight）**：边的数值属性，两城市间的距离

### 图的分类
#### 按方向性
- **无向图**：边没有方向，好友关系、道路网络
- **有向图**：边有方向，关注关系、工作流程

#### 按权重
- **无权图**：所有边权重相同，社交网络关系
- **有权图**：边有不同权重，地图距离、网络流量

#### 按连通性
- **连通图**：任意两点可达，交通网络
- **非连通图**：存在孤立点，社交群体

#### 按环
- **无环图**：没有环路，任务依赖、家族树
- **有环图**：包含环路，道路网络

### 图的表示方法
#### 1. 邻接矩阵
使用二维矩阵表示图：
- **存储方式**：matrix[i][j] 表示顶点 i 到 j 的边
- **检查边存在**：O(1)
- **添加/删除边**：O(1)
- **遍历邻接点**：O(V)
- **空间复杂度**：O(V²)
- **适用**：稠密图（边很多）

#### 2. 邻接表
为每个顶点维护一个列表：
- **存储方式**：数组 + 链表
- **检查边存在**：O(deg(v))
- **添加/删除边**：O(1)
- **遍历邻接点**：O(deg(v))
- **空间复杂度**：O(V+E)
- **适用**：稀疏图（边较少）

### 图的遍历算法
#### 广度优先搜索（BFS）
像"水波扩散"一样，从起点开始，先访问所有直接邻居，再访问邻居的邻居。

**算法步骤**：
1. 将起点放入队列并标记为已访问
2. 当队列不为空时：
   - 取出队首顶点 v
   - 访问 v 的所有未访问邻居，将它们放入队列并标记为已访问

**应用**：
- 无权图的最短路径
- 社交网络中的"六度空间"
- 网页爬虫

#### 深度优先搜索（DFS）
像"走迷宫"一样，选择一条路走到尽头，然后回溯，再走另一条路。

**算法步骤**（递归）：
1. 从顶点 v 开始，标记为已访问
2. 对于 v 的每一个未访问邻居 u：
   - 递归调用 DFS(u)

**应用**：
- 连通性检测
- 拓扑排序
- 强连通分量
- 迷宫求解

## 示例
### Python 实现图的遍历
```python
from collections import deque

def bfs(graph, start):
    """广度优先搜索"""
    visited = set([start])
    queue = deque([start])
    result = []
    
    while queue:
        vertex = queue.popleft()
        result.append(vertex)
        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return result

def dfs_recursive(graph, vertex, visited=None, result=None):
    """深度优先搜索（递归）"""
    if visited is None:
        visited = set()
    if result is None:
        result = []
    
    visited.add(vertex)
    result.append(vertex)
    
    for neighbor in graph[vertex]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited, result)
    return result

# 测试
test_graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'],
    'E': ['B', 'F'],
    'F': ['C', 'E']
}

print("BFS:", bfs(test_graph, 'A'))
# 输出：['A', 'B', 'C', 'D', 'E', 'F']

print("DFS:", dfs_recursive(test_graph, 'A'))
# 输出：['A', 'B', 'D', 'E', 'F', 'C']
```

## 易错点
1. **DFS 与 BFS 的混淆**
   - DFS 使用栈（递归），BFS 使用队列
   - DFS 适合找所有路径，BFS 适合找最短路径（无权图）

2. **图的遍历需要考虑图不连通的情况**
   - 可能需要从多个起点开始遍历
   - 使用循环检查所有顶点是否都被访问

3. **邻接矩阵与邻接表的选择**
   - 稠密图（边多）：邻接矩阵
   - 稀疏图（边少）：邻接表

4. **有向图与无向图的区别**
   - 无向图的边是双向的，矩阵对称
   - 有向图的边是单向的

5. **权重图的处理**
   - 邻接表中存储元组 (邻居，权重)
   - 邻接矩阵中存储权重值

3. 根据问题选择遍历算法（BFS/DFS）

5. 实现图的连通性检测

## 复杂度与考点
### 时间复杂度
- **BFS**：O(V+E)，V 为顶点数，E 为边数
- **DFS**：O(V+E)
- **邻接矩阵遍历**：O(V²)
- **邻接表遍历**：O(V+E)

### 空间复杂度
- **邻接矩阵**：O(V²)
- **邻接表**：O(V+E)
- **BFS/DFS**：O(V)（visited 集合 + 队列/栈）
